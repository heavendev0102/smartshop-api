## 1. INITIAL SETUP & ISSUES

### Issue 1: Alembic Migration Error - Missing psycopg2
**Error:**
```
ModuleNotFoundError: No module named 'psycopg2'
```

**Root Cause:**
- Alembic was trying to use psycopg2 for synchronous database operations
- But the app was configured to use asyncpg for async operations
- Mismatch between sync/async database drivers

**Solution:**
- Identified that the app uses `postgresql+asyncpg://` in DATABASE_URL
- Alembic was using `postgresql://` in alembic.ini
- Both configurations were correct, but psycopg2 wasn't installed

**Action Taken:**
- Installed `psycopg2` via pip (though asyncpg was already installed)
- This resolved the immediate import error

---

### Issue 2: Missing email-validator Package
**Error:**
```
ImportError: email-validator is not installed, run `pip install 'pydantic[email]'`
```

**Root Cause:**
- Pydantic's `EmailStr` field requires the optional email-validator package
- Package wasn't installed in the virtual environment

**Solution:**
- Installed `email-validator` package
- Alternative: Could have changed `EmailStr` to `str` if validation wasn't needed

---

### Issue 3: Incorrect datetime.timezone Reference
**Error:**
```
AttributeError: module 'datetime' has no attribute 'timezone'
```

**Location:** `app/models/user.py` lines 21-22
**Code:**
```python
created_date = Column(DateTime, default=datetime.timezone.utc)
modified_date = Column(DateTime, default=datetime.timezone.utc, onupdate=datetime.timezone.utc)
```

**Root Cause:**
- `timezone` wasn't imported from `datetime` module
- Incorrect reference to `datetime.timezone.utc`

**Solution:**
- Added `timezone` to imports: `from datetime import datetime, timezone`
- Changed to proper callable defaults:
```python
created_date = Column(DateTime(timezone=True), default=lambda: datetime.now(timezone.utc))
modified_date = Column(DateTime(timezone=True), default=lambda: datetime.now(timezone.utc), onupdate=lambda: datetime.now(timezone.utc))
```

### Issue 5: Pydantic Settings Configuration Error
**Error:**
```
pydantic_core._pydantic_core.ValidationError: 1 validation error for Settings
DATABASE_URL
  Field required [type=missing, input_value={}, input_type=dict]
```

### Issue 6: Missing Users Table in Database
**Error:**
```
sqlalchemy.exc.ProgrammingError: (psycopg2.errors.UndefinedTable) table "users" does not exist
```

**Root Cause:**
- Alembic migration `a9a5cb6504e2_create_users_table.py` existed but was empty

## 2. AUTHENTICATION & SECURITY IMPLEMENTATION

### User Registration Endpoint
**Endpoint:** `POST /api/v1/users/`
**Functionality:**
- Accepts `UserCreate` schema with user details
- Validates email uniqueness
- Hashes password using PBKDF2
- Creates user in database
- Returns user data

### User Login Endpoint
**Endpoint:** `POST /api/v1/users/login`
**Functionality:**
- Accepts `UserLogin` schema (email + password)
- Verifies credentials against database
- Generates JWT access token
- Returns token + user data

### Protected Endpoint Example
**Endpoint:** `GET /api/v1/users/me`
**Functionality:**
- Requires Bearer token authentication
- Validates JWT token
- Returns current user information

## 3. JWT TOKEN SYSTEM IMPLEMENTATION

### Security Module Enhancements
**File:** `app/core/security.py`

**JWT Configuration:**
```python
SECRET_KEY = "your-secret-key-here-change-in-production"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30
```

**JWT Functions:**
- `create_access_token(data: dict)` - Generates signed JWT token
- `verify_token(token: str)` - Validates and decodes JWT token
- `get_current_user()` - FastAPI dependency for authentication

### Token Response Schema
**File:** `app/schemas/user.py`
```python
class TokenResponse(BaseModel):
    access_token: str
    token_type: str = "bearer"
    user: dict
```

### Authentication Flow
1. User sends login credentials
2. System validates email/password
3. JWT token is generated with user ID and email
4. Token is returned to client
5. Client includes token in `Authorization: Bearer <token>` header
6. Protected endpoints validate token and extract user info

## 6. DATABASE SCHEMA

### Users Table Structure
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    first_name VARCHAR(100),
    last_name VARCHAR(100),
    username VARCHAR(100) UNIQUE,
    email VARCHAR(255) UNIQUE,
    phone_number VARCHAR(20),
    gender VARCHAR(10),
    date_of_birth TIMESTAMP WITH TIME ZONE,
    profile_image TEXT,
    password_hash TEXT NOT NULL,
    email_confirmed BOOLEAN DEFAULT FALSE,
    phone_number_confirmed BOOLEAN DEFAULT FALSE,
    is_active BOOLEAN DEFAULT TRUE,
    created_date TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    modified_date TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 7. SECURITY FEATURES IMPLEMENTED

### Password Security
- **Hashing Algorithm:** PBKDF2 with SHA256
- **Iterations:** 100,000 (industry standard)
- **Salt:** 32-character cryptographically secure random salt
- **Storage:** `salt$hash` format

### JWT Token Security
- **Algorithm:** HS256 (HMAC-SHA256)
- **Expiration:** 30 minutes
- **Claims:** User ID and email
- **Secret Key:** Configurable (change in production!)

### Authentication Flow
1. Password verification using constant-time comparison
2. JWT token generation with expiration
3. Bearer token validation on protected routes
4. Automatic user lookup from token claims
